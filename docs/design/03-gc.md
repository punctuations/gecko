# Garbage collector

## One per isolate

Each isolate has its own heap and its own collector, see 04-concurrency.md.
Isolates share no mutable objects, so a collector runs on a single thread inside
its isolate and never coordinates with another isolate's collector.

The one exception is the immutable shared buffers holding columnar data,
described below. They live outside the per-isolate heaps and are reference
counted.

## v0.0.1 collector

Precise, non-moving mark-sweep.

- Precise: the VM knows its roots exactly, meaning operand stacks, local slots,
  the globals table, and the live handles the host holds. NaN boxing makes
  values self-describing, so the collector can trace an object's fields from its
  type index.
- Non-moving: an object keeps its address for its whole life. That keeps FFI
  simple, because a pointer handed to C or Rust stays valid across a collection.
- Allocation: size-classed free lists over arena pages, with a bump pointer
  inside a fresh page.

A collection triggers when allocation passes a heap-size threshold, and that
threshold grows with the live size.

## Later

- Generational, around v0.0.5. A bump-allocated young arena with a copying minor
  collection, promoting survivors into the mark-sweep old space. Moving the
  young generation is safe because pointers visible to FFI are pinned or
  promoted.
- Incremental or concurrent marking, long-term, to bound pause times.

## Finalization

Finalization is neither prompt nor ordered. If `__del__` is supported at all, it
runs at an unspecified time during a collection, or not at all at shutdown. This
is a stated deviation from CPython, see 00-language.md. Clean resources up with
context managers rather than finalizers.

## Shared buffers

Immutable columnar buffers backing arrays and DataFrames are allocated outside
isolate heaps and reference counted. An actor message can transfer a handle to
such a buffer without copying its bytes. The buffer is freed when the last
handle drops. See 04-concurrency.md.
